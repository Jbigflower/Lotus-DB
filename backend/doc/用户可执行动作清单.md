# Lotus-DB 用户可执行动作清单

## 📋 概述

本文档详细描述了 Lotus-DB 系统中用户可以执行的所有主要动作，包括完整的调用链路、数据存储设计和技术实现细节。每个动作都遵循统一的架构模式：**前端 → API → Service → Repository → Database**。本文档和 Lotus-DB 前段页面设计相辅相成，互相补充。

## 🎬 电影管理模块

### 1. 搜索电影

**功能描述**: 支持关键词搜索和语义搜索，提供智能推荐

**调用链路**:

```
前端搜索框 → GET /api/movies/search
    ↓
MovieService.search_movies()
    ↓
MovieRepo.search() + ChromaRepo.semantic_search()
    ↓
MongoDB movies 集合 + Chroma 向量数据库
```

**存储设计**:

- **MongoDB**: `movies` 集合存储电影元数据
- **Chroma**: 存储电影描述、标题的向量表示
- **Redis**: 缓存热门搜索结果 (TTL: 1小时)

**数据流**:

1. 关键词搜索: MongoDB 文本索引查询
2. 语义搜索: Chroma 向量相似度检索
3. 结果合并: 权重算法排序
4. 缓存策略: Redis 存储查询结果

### 2. 添加电影

**功能描述**: 手动添加或通过 OMDB API 自动获取电影信息

**调用链路**:

```
前端表单 → POST /api/movies
    ↓
MovieService.create_movie()
    ↓
OMDBClient.fetch_metadata() → MovieRepo.create()
    ↓
MongoDB movies 集合 + Chroma 向量化
```

**存储设计**:

- **输入验证**: `MovieCreate` Pydantic 模型
- **数据存储**: MongoDB `movies` 集合
- **向量化**: 异步任务生成 embedding
- **静态资源**: 海报/背景图挂载到 `static/movies/{movie_id}/`

### 3. 编辑电影信息

**功能描述**: 修改电影元数据、标签、评分等信息

**调用链路**:

```
前端编辑表单 → PUT /api/movies/{movie_id}
    ↓
MovieService.update_movie()
    ↓
MovieRepo.update() + ChromaRepo.update_embedding()
    ↓
MongoDB 更新 + Chroma 重新向量化
```

**存储设计**:

- **乐观锁**: 使用 `updated_at` 字段防止并发冲突
- **软删除**: `is_deleted` 标记，保留历史数据
- **缓存失效**: Redis 相关缓存自动清理

### 4. 删除电影

**功能描述**: 软删除电影记录，保留关联资产

**调用链路**:

```
前端删除确认 → DELETE /api/movies/{movie_id}
    ↓
MovieService.delete_movie()
    ↓
MovieRepo.soft_delete() + AssetRepo.cascade_delete()
    ↓
MongoDB 软删除标记 + 关联资产处理
```

额外功能：支持回收站、支持彻底删除和恢复

## 📁 资产管理模块

### 5. 上传视频文件

**功能描述**: 上传电影视频文件，支持多种格式和质量

**调用链路**:

```
前端文件上传 → POST /api/assets/upload
    ↓
AssetService.upload_video()
    ↓
文件存储 + AssetRepo.create() + TaskService.create_processing_task()
    ↓
本地存储 + MongoDB assets 集合 + 异步处理任务
```

**存储设计**:

- **文件存储**: `videos/{library_id}/{当前library存储规范}/`
- **元数据**: MongoDB `assets` 集合
- **处理任务**: 异步生成缩略图、雪碧图、视频文件转码、视频文件字幕提取、视频声纹提取Hash……
- **版本管理**: 支持同一电影多个版本

### 6. 生成缩略图和雪碧图

**功能描述**: 自动生成视频预览图和时间轴雪碧图

**调用链路**:

```
后台任务触发 → TaskService.process_video()
    ↓
FFmpeg 处理 + AssetService.generate_thumbnails() + ……
    ↓
AssetRepo.update_assets()
    ↓
静态文件生成 + MongoDB 衍生资产记录
```

**存储设计**:

- **缩略图**: `static/movies/{movie_id}/{asset_id}/thumbnail.jpg`
- **雪碧图**: `static/movies/{movie_id}/{asset_id}/sprites/*.jpg`
- ……

### 7. 管理字幕文件

**功能描述**: 上传、编辑、同步字幕文件

**调用链路**:

```
前端字幕上传 → POST /api/assets/subtitles
    ↓
AssetService.upload_subtitle() → TaskService.process_subtitle()
    ↓
字幕解析 + AssetRepo.create() + ChromaRepo.index_subtitle()
    ↓
文件存储 + MongoDB 记录 + 字幕内容向量化
```

**存储设计**:

- **字幕文件**: `subtitles/{movie_id}/{language}.srt`
- **内容索引**: Chroma 存储字幕文本向量
- **时间轴**: 支持字幕时间戳搜索

## 👤 用户管理模块

### 8. 用户注册

**功能描述**: 新用户注册，创建个人媒体库

**调用链路**:

```
前端注册表单 → POST /api/users/register
    ↓
UserService.register()
    ↓
密码加密 + UserRepo.create() + LibraryService.create_default_library()
    ↓
MongoDB users 集合 + 默认媒体库创建
```

**存储设计**:

- **用户信息**: MongoDB `users` 集合
- **密码安全**: bcrypt 哈希加密
- **默认媒体库**: 自动创建个人媒体库

### 9. 用户登录

**功能描述**: 用户身份验证，生成访问令牌

**调用链路**:

```
前端登录表单 → POST /api/users/login
    ↓
UserService.authenticate()
    ↓
密码验证 + JWT 生成 + RedisRepo.store_session()
    ↓
JWT Token + Redis 会话存储
```

**存储设计**:

- **会话管理**: Redis 存储 JWT 令牌 (TTL: 24小时)
- **刷新令牌**: 长期有效的刷新令牌
- **登录历史**: MongoDB 记录登录日志

### 10.1 个人资产管理

**功能描述**: 管理用户创建的截图、笔记、剪辑等

**调用链路**:

```
前端资产操作 → POST/GET/PUT/DELETE /api/user-assets
    ↓
UserAssetService.manage_assets() + Optional[后台笔记向量化] 
    ↓
UserAssetRepo.crud_operations() + Optional[ChromaRepo.crub_oper]
    ↓
MongoDB user_assets 集合 + 文件存储 Chroma 用户笔记 集合
```

**存储设计**:

- **用户资产**: `user_assets/{user_id}/{asset_type}/`
- **权限控制**: 用户只能访问自己的资产
- **分类管理**: 按类型和创建时间组织

### 10.2 用户片单管理

**功能描述**: 管理用户创建的片单，实现片单的CRUD；管理用户的watchlist；管理用户的最爱（收藏）

**调用链路**:

```
前端片单操作 → POST/GET/PUT/DELETE /api/user
    ↓
UserService.manage_Movielist()
    ↓
UserRepo.crud_operations()
    ↓
MongoDB user 集合
```

**存储设计**:

- **权限控制**: 用户只能访问自己的favorites、watchlist、custom_lists

### 20. 通知模块

**功能描述**: 系统在任务完成、导入失败、扫描结束等事件发生时，向用户推送通知。

**调用链路**:

```
事件触发 (任务完成/错误/系统警告)
    ↓
NotificationService.send_notification()
    ↓
NotificationRepo.save() + Optional[SMTPClient/TelegramClient]
    ↓
Redis 存储通知记录 + 外部推送
```

**存储设计**:

- **Redis**: 短期存储系统通知，后期项目推进可以考虑使用 MongoDB
- **消息格式**: 标题、内容、类型（info/warning/error）、是否已读
- **推送方式**:
  - 前端 WebSocket 实时推送
  - 邮件通知（SMTP）
  - Telegram Bot（可选）
- **清理策略**: 支持按时间或数量限制自动清理旧通知

## 🔄 数据同步策略

### 核心原则

1. **最终一致性**: 多数据源之间保证最终数据一致
2. **异步处理**: 耗时操作异步执行，不阻塞用户操作
3. **缓存优先**: 热点数据优先从缓存读取
4. **故障恢复**: 支持数据恢复和重试机制

### 同步流程

```
用户操作 → MongoDB 主数据更新
    ↓
异步任务 → Chroma 向量更新
    ↓
缓存失效 → Redis 相关缓存清理
    ↓
通知机制 → WebSocket 推送更新
```

## 🚀 性能优化策略

### 数据库优化

- **MongoDB**: 合理索引设计，聚合查询优化
- **Chroma**: 向量维度优化，批量操作
- **Redis**: 合理的 TTL 设置，内存使用优化

### 应用层优化

- **异步编程**: FastAPI 异步特性充分利用
- **连接池**: 数据库连接池管理
- **批量处理**: 减少数据库交互次数
- **CDN 加速**: 静态资源 CDN 分发

**单元测试指导书**

**测试用例设计原则**

* 关注模块边界与解耦：优先测试 utils、core（中间件/异常/依赖）、services、repos 的纯函数与明确接口，再覆盖 routers（API）与 async_worker（后台任务）。
* 异步优先与稳定断言：统一使用 pytest-asyncio 与 httpx.AsyncClient；断言响应码、响应模型、响应头、日志内容和状态变化，避免时间/随机值带来的不确定性。
* 隔离外部依赖：单元测试用 Mock/Fake 替代 Mongo/Redis/LanceDB/Chroma/网络与文件系统等；将真实连接留给集成测试。
* 全路径覆盖与异常分支：对装饰器与异常映射（AppBaseException）覆盖成功与失败路径，验证日志关键字段（trace_id、action、layer）。
* 安全与权限：覆盖认证依赖 get_current_user 正常与失效分支；接口的 401/403 行为与 WWW-Authenticate: Bearer 头。

**测试环境配置说明**

* 依赖安装*   pip install pytest pytest-asyncio httpx pytest-cov fakeredis
* 选配（集成测试/复杂 mock）：pip install testcontainers motor pymongo lancedb chromadb pillow pymediainfo
* 目录结构*   tests/unit/：单元测试（全部 mock 外部依赖）。
* tests/integration/：容器或本地依赖的集成测试。
* tests/e2e/：端到端场景（API + 异步任务链路）。
* 顶层 tests/conftest.py：共享 fixtures（app、async_client、fake_redis、tmp_media_dir）。
* 运行配置*   pytest.ini 或 pyproject.toml：addopts = "--cov=src --cov-report=term-missing -q"
* 环境降噪：LOG_LEVEL=WARNING，必要时使用 caplog 捕获并断言日志。
* 关键依赖替换*   数据库初始化：将 src.core.database.init_databases/close_databases 替换为 no-op。
* Redis：get_redis_client 替换为 fakeredis.aioredis.FakeRedis()。
* 认证：app.dependency_overrides[get_current_user] = fake_user。
* 外部 IO：requests.get、subprocess.run、PIL.Image.open、MediaInfo.parse 以 monkeypatch/mock 控制。

**测试数据准备方法**

* Pydantic 工厂：为常用模型构造轻量数据，如 UserCreate、LibraryCreate、MovieCreate，统一采用最小必填字段。
* Faker/FactoryBoy（可选）：用 faker.seed(123) 保持可重复性；复杂嵌套结构使用工厂类。
* 文件与媒体*   通过 tmp_path 创建临时目录与文件；配合 resolve_library_root/ensure_dir_rw 验证路径逻辑。
* 图片用 PIL 生成简单图；视频相关命令用 monkeypatch subprocess.run 避免真实 FFmpeg。
* Redis/Mongo/LanceDB/Chroma*   单元层：注入 Fake/Stub 客户端或管理器对象；验证状态与异常映射。
* 集成层：testcontainers 启动容器，注入 DatabaseSettings 环境变量或 monkeypatch 返回的设置。

**测试执行步骤**

* 跑全部单测：pytest -q
* 带覆盖率：pytest --cov=src --cov-report=term-missing -q
* 过滤慢测试：pytest -q -m "not integration and not slow"
* 跑指定文件/用例：pytest -q tests/unit/test_utils_path.py -k "resolve_library_root"

**预期结果验证标准**

* API 路由*   状态码：正常 2xx，异常分支返回 4xx/5xx。
* 响应模型：与 response_model 字段一致（类型/必填）。
* 认证：无 token 返回 401，响应头含 WWW-Authenticate: Bearer；有 token 返回正确数据。
* 中间件：返回头含 X-Trace-ID；CORS/TrustedHost/GZip 行为合理（可用最小断言）。
* Core/Services/Repos/Logic*   装饰器：router_handler/service_handler/repo_handler/logic_handler 成功/失败日志均含 trace_id、action，异常映射到 AppBaseException；层级 layer 正确。
* 异常：register_exception_handlers 能将未捕获异常映射为统一 JSON，包含 error.type/code/layer。
* 事务与分页：Repo 事务上下文异常时回滚；分页 pages=(total+size-1)//size 正确。
* DB 管理器*   未初始化访问 client/db 抛 DBInitializationError；connect/close 切换 is_connected。
* 错误参数抛 DBConfigurationError/DBConnectionError/DBTimeoutError（通过 stub 抛错验证）。
* Utils*   路径：绝对路径、前缀路径、逻辑名解析正确；mask_library_root 返回相对逻辑名。
* 文件：生成临时文件 → 推断扩展名 → 确保唯一命名；异常时清理临时文件。
* 媒体：缩略图与雪碧图生成在 mock 环境下返回结构字段。
* 异步任务*   send_task/_invoke_task：不同函数签名（无参/单参/kwargs/同步）均能被正确调用或清晰报错。
* Worker：在 fake Redis 下消费消息、xack/xdel 成功、失败时按配置重试。

**测试覆盖率要求**

* 总体 ≥ 80%；关键路径模块 ≥ 90%。
* 关键模块（≥ 90%）*   src/core/exceptions.py、src/core/handler.py、src/core/middleware.py、src/core/dependencies.py
* src/utils/*
* 中等覆盖（≥ 80%）*   src/routers/*、src/services/*、src/logic/*、src/repos/*
* 可接受覆盖（≥ 70%）*   src/utils/media_processing.py（强依赖外部命令，多用 mock）
* src/async_worker/*（在 fake Redis 下覆盖核心流程）
* 覆盖率度量：pytest --cov=src --cov-report=term-missing；对确实不可测的分支可用 # pragma: no cover（保守使用）。

**各个模块的测试细则**

* Core*   main.py：通过 monkeypatch init_databases/close_databases 为 no-op；验证自定义 /docs 与 /health。
* middleware.py：断言响应头包含 X-Trace-ID；用 caplog 捕获“请求开始/完成/异常”日志。
* exceptions.py：触发未处理异常并验证统一响应结构与 layer 字段。
* docs.py：custom_openapi 追加 components.securitySchemes.BearerAuth；/docs 返回自定义 UI。
* dependencies.py：无效 token 抛 401 且头部包含 WWW-Authenticate: Bearer。
* Routers*   core/routers.py：四类路由均被注册（Users/Auth/Libraries/Movies）。
* routers/auth.py：/auth/login 成功返回 access_token/token_type/user；异常由 service_handler 映射（mock service.login）。
* routers/users.py：依赖覆盖 get_current_user；覆盖 activate/deactivate 等路径。
* routers/libraries.py、routers/movies.py：覆盖查询/新增/分页/上传分支；文件上传用 UploadFile 与临时目录。
* Services*   users/auth_service.py：成功与失败路径；验证 performance_monitor 日志（caplog 或检查性能日志文件增加）。
* 其他服务：mock 对应 Logic，验证输入/输出封装与异常映射。
* Logic*   典型方法与异常分支；失败时 raise_with_context(layer="Logic", context={...})。
* Repos*   BaseRepo：在 mock AsyncIOMotorCollection 下验证 insert_many/find_many/update_by_id/count、软删除过滤、分页统计。
* users/user_repo.py：search_users 文本搜索、角色/状态过滤与分页结果。
* DB*   Mongo/Redis/LanceDB/Chroma 管理器：connect/close/is_connected 状态机；错误路径抛对应异常；未初始化访问属性抛 DBInitializationError。
* Async Worker*   core._invoke_task 覆盖多签名与异常；send_task 写入 stream 字段完整。
* Worker：用 fake Redis 发布消息后 start() 消费、确认与删除；失败重试策略基本路径。

**测试报告编写规范**

* 报告结构*   概述：测试范围、版本与环境（依赖/容器）、执行命令。
* 覆盖率：总体与模块分布，列出低覆盖模块与原因、提升建议。
* 缺陷与失败用例：现象、日志片段、复现步骤、影响评估。
* 风险与后续计划：外部依赖策略、待补充用例与集成测试排期。
* 呈现标准*   路径/命令统一使用反引号：如 tests/unit/...、pytest -q。
* 附件：关键日志、覆盖率 term-missing 输出、示例截图（如中间件头部、Swagger UI）。
* 质量指标*   用例通过率、平均执行时长、慢用例列表（标记 slow）、不稳定用例（标记与重试策略）。

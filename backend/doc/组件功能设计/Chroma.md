Chroma 在你的系统中主要负责 **向量化存储与相似性检索**，与数据库形成互补：

- **向量化目标**：
    
    - 影片信息：标题 + 描述 + 标签
        
    - 用户笔记
        
    - 字幕内容
        
- **核心能力**：
    
    1. **向量化**
        
        - 将文本/多媒体信息转换为向量（embedding）
            
        - 可配合 LLM/Embedding 模型生成向量
            
    2. **向量存储与管理**
        
        - 高速向量存储
            
        - 支持批量插入、更新、删除
            
    3. **相似性检索**
        
        - 根据向量搜索最相似条目（KNN）
            
        - 支持阈值、Top-K 返回
            
    4. **与 MongoDB 同步**
        
        - 例如当用户笔记/影片信息更新时，向量库也更新对应向量
            
    5. **后台任务接口**
        
        - 支持异步任务向量化
            
        - 可与 Celery / Task 层结合，处理大批量数据
            
    6. **Agent 支持**
        
        - 为 RAG / 智能检索提供统一接口
            
        - Service 层或 Agent 可直接调用 Chroma 接口进行向量检索

```
Router/API --> Service --> Repo --> DB/Chroma
```

| 层            | Chroma 相关职责                                                      |
| ------------ | ---------------------------------------------------------------- |
| DB 层         | 提供 Chroma client 实例、连接管理、集合获取、事务支持（基础功能）                         |
| Repo 层       | 封装向量操作接口：插入、更新、删除、批量操作、检索。保持与 Mongo 数据同步策略接口（但不做业务逻辑）            |
| Service 层    | 处理业务逻辑：例如“用户笔记更新 → 向量化并同步”，“影片检索 → 调用 Chroma 检索 + 聚合 Mongo 数据返回” |
| Router/API   | 接收请求，调用 Service，返回结构化结果                                          |
| Agent/智能体    | 可直接调用 Service 或 Repo 提供的向量接口，实现 RAG、问答、推荐等智能功能                   |
| 任务层 / Celery | 调度批量向量化任务，如电影批量导入、字幕向量化、用户笔记向量化                                  |

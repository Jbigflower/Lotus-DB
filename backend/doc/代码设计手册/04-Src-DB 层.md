## 一、整体思路

**DB 层（src/db/）主要目标**：提供**底层连接和基础操作能力**，使上层 Repo/Service 层能够专注于业务逻辑，而无需关心连接管理、客户端初始化或事务处理细节。

**核心功能**：
1. **初始化连接**
    - MongoDB / Redis / Chroma 客户端实例化
    - 连接池管理
    - 异步/同步支持
        
2. **获取 client / db / collection**
    - 提供统一接口供 Repo 调用
    - 可封装一些基础方法（如获取集合对象 `get_collection("movies")`）
        
3. **关闭连接 / 清理资源**
    - 应用 shutdown 时释放资源
        
✅ **原则**：DB 层不应包含业务逻辑或具体操作（如插入特定 Movie、检索某向量），这些属于 Repo 层职责。

---
MongoDB 示例 (`mongo.py`) 功能目标：
提供一个统一、全局可复用的 MongoDB 连接实例，并封装基础操作。具体功能包括：
- 单例模式创建 MongoDB 客户端
- 连接池管理（确保高并发下性能稳定）
- 提供 `get_db()` 和 `get_client()` 方法，供 Repo 层直接使用
- 提供 `init_db()`和 `close_db()`方法，供 `core` 层快速封装数据库连接
- 异步支持（`motor` 异步 MongoDB 驱动）
- 支持事务上下文管理器
- 支持索引创建与初始化（可在启动时创建高频访问字段索引）

## 二、设计要求

在实现 DB 层时，应遵循以下原则：
- **配置与日志统一**
    - 使用 Configs 层提供的配置工具和日志工具
        
- **字段与模型复用**
    - 使用 Src-Models 层提供的字段定义
    - 若需新增字段，应在保持抽象性和普适性的前提下，在 Src-Models 相应文件中追加定义
        
- **职责边界明确**
    - DB 层仅负责“能否连接数据库及基础操作”
    - Repo 层负责“如何操作具体数据”

数据表设计及索引策略：
- movies：核心元数据、高频访问字段聚合（text 索引 + year/type 等复合索引）
- assets：多源资源、衍生资源、版本管理（movie_id + source 唯一索引）
- libraries：媒体库配置，统一存储规则（library_name 唯一索引）
- user：基础信息 + 高频收藏/片单内嵌（username/email 唯一索引）
- user_assets：用户个人创作内容（user_id + asset_id 复合索引）
- watch_history：长尾数据（user_id + movie_id 索引，可 TTL 或分片）
- tasks：后台任务（status + created_at 索引）
- logs：系统日志（时间 + 级别索引，分库分表优化）

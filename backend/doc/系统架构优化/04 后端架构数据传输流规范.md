### 目的

* **类型安全** ：避免每一层手动处理 dict → 模型 → dict，防止类型错误。
* **边界清晰** ：每一层只关注自己的逻辑和类型，不依赖外层的实现。
* **可维护性** ：接口签名明确，方便 IDE 自动补全和静态检查。

### 原则

* Repo 层只返回 BaseModel，主要为 InDB、Read、PageResult
* Logic 层处理业务逻辑，返回 BaseModel，并进行 InDB -> Read 转换
* Service 层负责接口级 DTO（Data Transfer Object）或直接传递 BaseModel 给 Router。
* Router 层由 Fastapi 自动格式转换

### 关于 exclude=True 在哪里调用

repo 层，convert 有一个 `dicts = [d.model_dump(exclude_unset=True) for d in data_list]` 保底补救
